### Scripts

#### `scripts/update_data.py`
```python
"""
Incremental data update script.
Updates existing CSV with latest quarters only.
"""
import pandas as pd
from sec_data_extractor import build_quarterly
from datetime import datetime

# Load existing data
df_existing = pd.read_csv("sec_quarterly_raw_data.csv")

# Get last date
last_year = df_existing['fy'].max()

# Fetch only new data
df_new = build_quarterly(
    tickers=df_existing['ticker'].unique().tolist(),
    start_year=last_year,
    end_year=datetime.now().year
)

# Append and deduplicate
df_combined = pd.concat([df_existing, df_new]).drop_duplicates(
    subset=['ticker', 'fy', 'fq']
)

# Save
df_combined.to_csv("sec_quarterly_raw_data.csv", index=False)
print(f"Updated with {len(df_new)} new observations")
```

## Setup Instructions

### 1. Initialize Repository
```bash
mkdir sec-earnings-analyzer
cd sec-earnings-analyzer
git init
```

### 2. Create Files
Create all files according to the structure above.

### 3. Add Content
Copy your code into respective files:
- Main script → `sec_data_extractor.py`
- Dashboard → `dashboard/streamlit_app.py`
- Documentation → Various `.md` files

### 4. First Commit
```bash
git add .
git commit -m "Initial commit: SEC Earnings Growth Analyzer"
```

### 5. Create GitHub Repository
1. Go to github.com
2. Click "New repository"
3. Name: `sec-earnings-analyzer`
4. Public repository
5. Don't initialize with README (you already have one)

### 6. Push to GitHub
```bash
git remote add origin https://github.com/yourusername/sec-earnings-analyzer.git
git branch -M main
git push -u origin main
```

## Best Practices

### What to Include in Git
✅ Source code
✅ Documentation
✅ Requirements.txt
✅ Sample data (small file)
✅ Configuration templates

### What NOT to Include
❌ Large CSV files (>50MB) - use Git LFS or separate hosting
❌ API keys / secrets
❌ Virtual environments
❌ IDE-specific files
❌ Generated outputs (unless for demo)

## Documentation Tips

1. **README.md**: Focus on "what" and "why"
2. **METHODOLOGY.md**: Explain "how" in technical detail
3. **Code comments**: Explain "why" (not "what" - code shows that)
4. **Docstrings**: API documentation
5. **Notebooks**: Show examples and insights
